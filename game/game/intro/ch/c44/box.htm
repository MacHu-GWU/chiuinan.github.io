資訊：MAP資料檔結構（每個MAP檔有10關資料）：

檔頭：5 byte，固定FD FB 3C 00 00
資料總長度：2 byte
低位元組壓縮資料長度：2 byte
低位元組壓縮資料
高位元組壓縮資料長度:2 byte
高位元組壓縮資料

壓縮方式採用Run Length：

  C = 讀取1 byte
  N = (C最低7位元值) + 1
  if (C最高位元 = 1) 讀取N byte做為資料
  else 讀取1 byte，重複N次做為資料

解壓後，將低位元組與高位元組合併在一起，便是未壓縮前資料，例如LNK20.MAP解壓後的資
料為：

0C 21 04 00 03 00 02 00 03 00 03 00 02 00 02 00
02 00 04 00 03 00 00 00 00 00 00 00 00 00 00 00

從位置0002h開始的10個字組，分別表示10關應顯示的大小，1最小，4最大。各關卡的內容資
料所在位置，假設關卡編號為X（0～9，例如194關便是3），關卡資料由左上依水平方向往右
計算為第i個（由0開始），則它的實際資料所在位置為（每個資料2 byte）：

  039Eh + X*2 + i*22

資料值的意義如下：

  00 00 無
  01 00 牆壁
  02 00 箱子
  03 00 通道
  04 00 目的地
  05 00 人物初始位置

操作：F1 = 特殊功能
